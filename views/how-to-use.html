    應用程式概念

    一站式管理達人的Facebook及Slack個人品牌修煉頻道，促進交換社交貨幣。

    方便達人檢查有沒有漏看分享到品牌修煉的Facebook帖子，覆核是否有依照品牌修煉的規則回Slack頻道Mark Liked。為達人提供發佈頻率與粉絲回應的統計及分析數據。

    新功能 - 粉絲分析助手

    登入後，從Menu Bar按「粉絲分析助手」的連結，就會進入如下圖的網頁。預設顯示為「過去7天」的統計記錄。你可以在表格的右上方更變日期範圍選項，然後按「取得分析」向伺服器查詢更多天數的數據記錄。

    使用說明

    登入並獲取Slack授權

    1. 按下「Sign in with Slack」按鈕，瀏覽器會轉至Slack的登入網頁。
    2. 填入coxohk.slack.com，登入達人圈子的Workspace。
    3. 然後填寫你的電郵地址與密碼，登入你的Slack帳號。
    4. 確認連結到達人圈子，然後按下「Continue」。
    5. 選擇Post to [品牌修煉的Channel]，授權COXOAPP讀取私人頻道及修改Emoji反應。
    6. 當程式確認Slack的授權後，會自動轉入應用程式的主畫面。
    頁面頂部

    抓取「品牌修煉」分享的Facebook貼文

    1. 進入主畫面後，應用程式會自動抓取Slack之中的「品牌修煉」頻道內訊息記錄，預設為最新15個記載。顯示搜索到的Facebook貼文。
    2. 如果你尚未登入Facebook帳號，會看見「使用Facebook帳戶登入」的按鈕。請按下，按照Facebook的指示，使用你的Facebook帳戶繼續。
    3. 如果你想抓取更多Slack記錄，可以修改輸入方塊之中的數字，設定上限為100，然後按「確定」按鈕。
    4. 取得Slack資料後，會顯示搜索到的Facebook貼文總數，以及已經在Slack回應了的數目。
    5. 顯示灰色背景的貼文，表示該篇貼文在Slack上已留了回應。在嵌入式貼文下方之「標記到Slack」按鈕旁亦會顯示綠色的 號。
    頁面頂部

    顯示/隱藏已標記到Slack(灰色背景)的貼文

    1. 按下「」按鈕，應用程式就會把有灰色背景的嵌入式貼文顯示起來。
    2. 按下切換成「」的按鈕，應用程式就會把已隱藏起來的嵌入式貼文再次隱藏出來。
    頁面頂部

    在Facebook嵌入式貼文「讚好」

    1. 按下Facebook嵌入式貼文的「讚好」按鈕。
    2. 如果看到「確認」按鈕，請再次按下，然後在彈出的視窗確認「讚好」。


      (為什麼在點擊嵌入式Facebook貼文的「讚好」按鈕後會看到「確認」按鈕？)

    頁面頂部

    在Slack回應「讚好」

    1. 按嵌入式貼文下方的「標記到Slack」按鈕。
    2. 如果成功在「品牌修煉」頻道的對應記錄上留下反應，「標記到Slack」按鈕旁就會出現綠色的 號。如果失敗，則會出現紅色的 。

    常見問題

    1. 為什麼在點擊嵌入式Facebook貼文的「讚好」按鈕後會看到「確認」按鈕？

      • Facebok 常見問題 - 當有特定用戶人數分別確認讚好之後，該確認步驟將不再出現。

    2. 出現錯誤信息

      • 如果沒有該Channel的讀取權限，則會顯示如下圖的錯誤信息。

    3. 為什麼不設計成自動同步Facebook與Slack之間的「讚好」？

      • 基於2018年2月份的Facebook隱私政策修定，現在已經不再提供API方法去傳取「讚好」貼文的用戶資料。
        更無法經由其API接點，對用戶貼文添加「讚好」反應。詳細請參考：Facebook Graph API 說明 (英文)。
